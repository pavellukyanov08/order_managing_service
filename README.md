# Приложение для работы с кошельками пользователей через систему авторизации JWT (FastAPI)

Данный проект реализует управление кошельками пользователей через систему авторизации основе JWT.

*Функционал логаута реализован не до конца, планирую крикрутить `redis` для колноценной работы*

## Функциональность

### 1. Модуль взаимодействия с пользователями

#### Регистрация (create_user)
Пользователь создаёт аккаунт, указав:
- имя / фамилию / отчество
- email
- пароль + подтверждение пароля

Пароль хэшируется и сохраняется в базе данных.

#### Авторизация (Login)
Вход по email и паролю.  
Система выдаёт пару токенов:
- **access token (JWT)** — короткоживущий
- **refresh token (JWT)** — используется для обновления access токена

#### Идентификация пользователя
Каждый защищённый запрос проходит:
1. извлечение токена из заголовков
2. декодирование JWT
3. получение пользователя

#### Logout (хочу реализовать в redis)
При выходе:
- текущий access-токен отменяется и удаляется из redis
- токен становится недействительным
- дальнейшие запросы с этим токеном → 403 Forbidden

#### Получение кошелька
После авторизации пользователь может запрашивать информацию о своей кошельке:
- операции по кошельку, баланс

#### Изменение кошелька
Пользователь может управлять кошельком, производя начисление или списание

*сделана защита от нулевого баланса и списания большей суммы, чем на балансе*

#### Мягкое удаление аккаунта
При удалении:
- устанавливается `is_active = False`
- выполняется logout (после добавления redis)
- вход в аккаунт становится недоступным


## 2. Система разграничения прав доступа

Система содержит модель ролей и разрешений.

### Роли:
### `admin`: 
Имеет доступ ко всему

### `user`: 
может только получать информацию о себе и менять свои данные

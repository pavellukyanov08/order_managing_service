# Приложение для обработки заказов клиентов с авторизацией через систему авторизации JWT

Данный проект реализует управление взаимодействие пользователей с заказами через систему авторизации основе JWT.

## Функциональность

### 1. Модуль взаимодействия с пользователями

#### Регистрация (create_user)
Пользователь создаёт аккаунт, указав:
- имя / фамилию / отчество
- email
- пароль + подтверждение пароля

Пароль хэшируется и сохраняется в базе данных.

#### Авторизация (Login)
Вход по email и паролю.  
Система выдаёт пару токенов:
- **access token (JWT)** — короткоживущий
- **refresh token (JWT)** — используется для обновления access токена

#### Идентификация пользователя
Каждый защищённый запрос проходит:
1. извлечение токена из заголовков
2. декодирование JWT
3. получение пользователя

#### Logout
При выходе:
- текущий access-токен отменяется и удаляется из redis
- токен становится недействительным
- дальнейшие запросы с этим токеном → 403 Forbidden

## 2. Система разграничения прав доступа

Система содержит модель ролей и разрешений.

### Роли:
### `admin`: 
Имеет доступ ко всему

### `user`: 
может только получать информацию о себе
